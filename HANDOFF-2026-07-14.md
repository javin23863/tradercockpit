## 3. THE BUG + root fix (most important thing to understand)

**Symptom (operator):** "instagram aspect ratio … don't match frames."
**Diagnosis:** reels were true 9:16, but the **news-article b-roll had ~8% black bars baked in**
(measured on the live IG video: 54px top / 52px bottom of 1280).
**Root cause:** `tools/visuals/fetch_news_shots.mjs` → `kenBurns()` used
`scale=...:force_original_aspect_ratio=decrease` + `pad=...:black` — a **fit-and-pad**, which
letterboxes any article crop wider than 16:9. Those bars flowed into the master and into every 9:16 short.
**Fix (permanent):** **cover-and-crop** — `force_original_aspect_ratio=increase` + `crop=3840:2160`.
Also added **`--reuse-png`** so news visuals can be re-rendered from cached `news-work/*.png` with **no
re-fetch** (deterministic, no article drift).

**Verify contract (use this every build):**
- every news visual → `cropdetect` reads `1920:1080:0:0`
- every short → `1080:1920:0:0` **and** `ffprobe` reads `1080,1920,1:1,9:16`

**Re-render recipe used (repeatable):**
```
rm visuals/{02-news-blockade,04-news-transits,09-why-hormuz,10-news-supply,12-week-ahead}.mp4
node tools/visuals/fetch_news_shots.mjs --reuse-png tools/visuals/news-hormuz-v4.json productions/video-02-hormuz-v4
```

## 4. Hard-won lessons (don't relearn these)

1. **`produce.py --stage assemble` caches parts** (`if not part.exists()`, line 135). Changed a visual?
   **Delete its `build/part-NN.mp4`** or the change silently won't land. Multi-visual sections → `part-NN-J.mp4`.
2. **YouTube titles reject `>`** (`invalidTitle`). Killed one short upload; retitle without it.
3. **`channel_optimize.py` has no argparse — it RUNS on import.** `--help` executed a live push.
4. **`watermarks().set()` crashes httplib2** (`range() arg 3 must not be zero`) — the request is sent
   but the response fails to decode. **YT corner watermark is NOT confirmed set** → do it by hand:
   Studio → Customization → Branding → upload `tools/visuals/brand/watermark.png`.
5. **Never copy the live Chrome profile** to get CDP on logged-in sessions — correctly blocked as
   credential theft. Chrome also refuses `--remote-debugging-port` on the default profile.
   **Right pattern:** debug Chrome on a *fresh* profile + operator logs in **once**.
6. **Blind GUI automation (SendKeys/coords) is unreliable** — it typed a URL into Instagram's search
   box. Use **CDP + DOM/aria selectors + screenshot-verify** every step.
7. **TikTok pre-fills the caption with the FILENAME** — select-all + Backspace before typing.
8. **Meta page tokens expire in hours** because the long-lived exchange fails on a **stale
   `META_APP_SECRET`**. Refresh by pulling the page token from `/me/accounts` with a fresh user token.
9. **FB Page profile picture + cover ARE API-settable** (`POST /{page}/photos` → `?cover=`, and
   `POST /{page}/picture`) — corrects an earlier "manual-only" note. **Category is NOT** (UI only).

### Round-2 scars (2026-07-14, the re-replace)
10. **A vertical can be 1080×1920 / DAR 9:16 and still be WRONG.** The container check passes on an
    anamorphic squeeze. The real check is *content*: TradingView price axis visible, candles not thin.
    Always eyeball a frame grid; ffprobe alone is not a verification.
11. **`produce.py stage_shorts` omits `CLIP_ANCHOR=right`** → center crop kills the price axis. The
    shorts recipe of record is `node tools/handoff/recut_shorts.cjs` (right-anchor + article treatment).
12. **News-article b-roll can't take a 9:16 crop at all** — text is cut both sides. recut_shorts now
    luma-detects article windows (`signalstats` YAVG at 2fps, threshold **160**; globe terrain plateaus
    ~128, articles ~200-227) and renders them as **fit-width over a blurred/darkened background**,
    time-gated with `enable='between(t,..)'`; charts keep the right-anchor crop. SRT burned last.
13. **TikTok Studio row menus need a TRUSTED hover chain AND a zoomed-out page.** `element.click()`
    opens the menu but leaves it unpositioned/invisible, so coordinate clicks miss; and at 100% zoom the
    actions column is off-screen right. Recipe: `document.body.style.zoom="0.6"` → mouse.move(row) →
    mouse.move("…") → click → mouse.click(Delete) → click confirm. (`tools/handoff/tt_delete4.cjs`)
14. **TikTok prefills the filename into the caption LATE** — clearing once races the prefill and ships
    `clip-00N….vertical` as the caption (happened to 4 of 5 posts; had to delete + repost). Clear in a
    verify-retry loop until the box reads empty, then type, then re-read the caption before Post.
15. **YouTube `videos().update` needs the `youtube` scope** (`token_channel.json`); `token.json`
    (upload-only) 403s. Uploads = 1600 quota units each.
