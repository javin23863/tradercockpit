# 🎬 SKILL — Props
_prop sheets, depth/volume, controlling the exact action. Apply EVERY video. Rules compound as we study more references._

## from: Seedance 4K hyperreal AI short film breakdown (Adil) (2026-07-05T17:43:42.641Z)
- Treat prop sheets exactly like character sheets: gray background, multiple views, and every required detail (labels, buttons, text) enumerated in the prompt.
- Reject flat, shadow-free prop renders — a prop with no depth cues will animate like a 2D decal.
- Approve a prop only when it shows physical volume: tapered/ergonomic edges, cast shadows, soft studio reflections.
- If one image model keeps producing flat props, switch models (GPT Image 2.0 → Nano Banana Pro fixed depth and highlights instantly) instead of re-prompting endlessly.
- To control the exact action on a prop (e.g., which button gets pressed), draw a red arrow directly onto the prop sheet pointing at the target, then feed that annotated image to the prompt assistant.

## from: AI commercial/ad workflow (Adil) (2026-07-05T17:53:13.981Z)
- Give every prop that repeats across scenes its own clean reference sheet.
- Skip motion/performance notes on props — they're objects, they don't perform; the sheet alone is enough.
- In tight close-ups, lock each drift-prone prop (drink, mug) with one clean reference so it doesn't change cut to cut.
- Pin a prop's size and position on the schematic map (e.g., "2x a person's height, locked to the hydrant's right") and restate hard locks in every prompt ("lock the backpack on both shoulders in every single cut").

## from: Hell Grind AI feature film — 28 pro tips (Higgsfield) (2026-07-05T17:56:56.909Z)
- Build a dedicated PROP SHEET for any prop the model misuses, showing multiple sides — a monster's claw kept being held like a sword until a sheet showed the outside AND inside of the hand.
- When a prop is repeatedly mishandled, treat it as a missing-reference problem: generate a better prop sheet before rewriting the prompt again.
- Specify which prop or weapon each character holds in every cut of the beat-by-beat action script so the model can't substitute or invent one.
- Keep prop states on the canvas alongside character states — held, dropped, or missing per scene.

## from: Reference clips — ultra-realism prompt structures (2026-07-05T18:02:10.690Z)
- Treat any screen within the frame (phone, monitor, tablet) as a controlled prop: describe what is displayed on-screen AND what is happening in real space simultaneously so both stay in sync.
- Specify the physical handling of props in motion ("arms stretched up," "arms gradually lower as he follows the falling object," "phone tilting from pointing straight up to pointing forward") so the model animates the grip, not just the object.

## from: Hell Grind Ep2 — full unedited AI-film process (Adil): style-prefix, 44-iteration locations, diagnosis loop, team collab (2026-07-05T19:07:32.444Z)
- Give props their own reference sheets rather than describing them ad hoc.
- Keep simple props simple: "closeup of a handwritten note on a blue sticky note that reads 'food in the fridge'" — quote the exact text.
- Never generate a multi-face prop (photo wall) in one image — faces drift; generate each photo individually, then stitch them together in Photoshop.
- Attach the location as a shared reference so props sit in the right space, but instruct the assistant NOT to re-describe the location in every prop prompt (it bloats them all).
